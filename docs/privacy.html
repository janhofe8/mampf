Datenschutzerklaerung

So gehen wir mit deinen Daten um.

Zuletzt aktualisiert: Maerz 2026

🔍 Ueberblick

MAMPF ist ein persoenlicher Restaurant-Guide fuer Hamburg. Deine Privatsphaere ist uns wichtig. Wir erheben nur die minimal notwendigen Daten, um die App-Funktionen bereitzustellen. Es gibt kein Tracking, keine Werbung und keine Weitergabe deiner Daten an Dritte.

Kurzfassung: Wir speichern eine anonyme Geraete-ID und deine Community-Bewertungen. Sonst nichts. Kein Tracking, keine Analytics, keine Weitergabe an Dritte.

📊 Welche Daten werden erhoben?

Daten Zweck Speicherort
Anonyme Geraete-ID (UUID) Zuordnung deiner Community-Bewertungen zu deinem Geraet Lokal auf deinem Geraet + Supabase Server
Community-Bewertungen (1-10) Anzeige der Community-Durchschnittsbewertungen Supabase Server
Standort (GPS) Anzeige nahegelegener Restaurants auf der Karte Wird nicht gespeichert — nur lokal auf dem Geraet verwendet

🔐 Geraete-ID (Device ID)

Beim ersten Start der App wird eine zufaellige, anonyme Geraete-ID (UUID) erstellt. Diese ID enthaelt keine persoenlichen Informationen und kann nicht zu dir als Person zurueckverfolgt werden.

Die Geraete-ID dient ausschliesslich dazu, deine Community-Bewertungen deinem Geraet zuzuordnen, sodass du deine eigenen Bewertungen bearbeiten und loeschen kannst.

  • Die ID wird lokal auf deinem Geraet gespeichert
  • Die ID wird zusammen mit deinen Bewertungen auf dem Server gespeichert
  • Es handelt sich um eine zufaellig generierte UUID — kein Name, keine E-Mail, kein Account

⭐ Community-Bewertungen

Wenn du ein Restaurant bewertest, wird deine Bewertung (eine Zahl von 1 bis 10) zusammen mit deiner anonymen Geraete-ID auf unseren Servern gespeichert. Aus allen Bewertungen wird der Community-Durchschnitt berechnet, der in der App angezeigt wird.

  • Nur die numerische Bewertung wird gespeichert (keine Texte, keine Kommentare)
  • Die Bewertung ist nur ueber die anonyme Geraete-ID zugeordnet
  • Du kannst jede Bewertung jederzeit in der App aendern oder loeschen

📍 Standortdaten

MAMPF fragt nach deinem Standort, um dir Restaurants in deiner Naehe auf der Karte anzuzeigen. Dein Standort wird ausschliesslich lokal auf deinem Geraet verwendet und zu keinem Zeitpunkt an unsere Server uebertragen oder gespeichert.

Die Standortfreigabe ist optional. Die App funktioniert auch ohne Standortzugriff — du siehst dann einfach alle Restaurants ohne Entfernungsangabe.

🚫 Was wir NICHT tun

  • Kein Tracking: Wir verwenden keine Analytics-Tools, keine Tracking-Pixel, kein Nutzerverhalten-Monitoring
  • Keine Werbung: Die App enthaelt keine Werbeanzeigen
  • Keine Weitergabe: Deine Daten werden nicht an Dritte verkauft, geteilt oder weitergegeben
  • Keine Accounts: Es gibt keine Registrierung, kein Login, keine E-Mail-Adressen
  • Keine Cookies: Die App verwendet keine Cookies oder vergleichbare Tracking-Mechanismen

🌐 Google-Daten

MAMPF zeigt Google-Bewertungen und -Rezensionszahlen als Referenzwert an. Diese Daten stammen aus der Google Places API und werden mit entsprechender Quellenangabe dargestellt.

Es werden keine Google-Nutzerdaten erhoben. Die angezeigten Google-Bewertungen sind oeffentlich zugaengliche Durchschnittswerte.

💾 Datenspeicherung

Die Daten werden auf Servern von Supabase gespeichert, einem Datenbankdienst mit Sitz in den USA. Supabase verwendet PostgreSQL-Datenbanken mit aktivierter Row-Level Security (RLS).

  • Bewertungen werden dauerhaft gespeichert, bis du sie loeschst
  • Die Geraete-ID wird lokal auf deinem Geraet gespeichert und verbleibt dort
  • Es werden keine Backups deiner persoenlichen Daten an Dritte weitergegeben

🗑 Daten loeschen

Du kannst all deine Daten jederzeit loeschen:

In der App: Oeffne die App-Einstellungen und tippe auf „Alle meine Daten loeschen". Damit werden alle deine Community-Bewertungen vom Server und die Geraete-ID von deinem Geraet entfernt.

Nach dem Loeschen werden deine Bewertungen nicht mehr in die Community-Durchschnitte eingerechnet. Beim naechsten Start der App wird eine neue, zufaellige Geraete-ID erstellt.

👶 Kinder

MAMPF richtet sich nicht an Kinder unter 13 Jahren. Wir erheben wissentlich keine Daten von Kindern. Da die App keine persoenlichen Daten abfragt (kein Account, keine Registrierung), ist eine versehentliche Erhebung von Kinderdaten unwahrscheinlich.
